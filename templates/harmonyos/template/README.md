## Repository Layout

- `AppScope/app.json5`: application-level HarmonyOS metadata
- `entry/`: installable HarmonyOS entry module
- `entry/src/main/module.json5`: module, ability, device, and page routing metadata
- `entry/src/main/ets/entryability/`: Stage model ability lifecycle
- `entry/src/main/ets/pages/`: ArkUI pages
- `scripts/`: bootstrap and local environment checks

## Getting Started

```bash
make bootstrap
make doctor
make build
make test
```

`make doctor` checks Node.js, ohpm, hvigor, and common HarmonyOS SDK environment variables before you spend time on a build.
Open the project with DevEco Studio when you need signing, device management, previewer, or emulator workflows.

## Notes

- `oh-package.json5` is the project dependency manifest.
- `build-profile.json5` declares the app signing placeholder and entry module.
- The generated starter intentionally keeps the first screen small: one entry ability and one ArkUI page.
- Signing is left to DevEco Studio or CI-specific `local.properties` configuration.
- Keep `bundleName`, module name, and SDK versions aligned with your target HarmonyOS release channel before store packaging.
